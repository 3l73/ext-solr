The Index Queue manages content to be indexed. Content enters the Index Queue by initializing below or when new content is created based on configuration. Items in the Index Queue are indexed newest changes to oldest changes until all items in the queue have been indexed.
  • Initialization
  • Queue
  • Log
{indexQueueInitializationSelector} Initializing the Index Queue is the most complete way to force re-indexing, or to build the Index Queue for the first time. The Index Queue Worker scheduler task will then index the items listed in the Index Queue.
